## 案例13：识别信用卡异常消费模式
**业务场景**：银行风控系统需要实时识别可能存在盗刷风险的信用卡交易。  
**SQL书写逻辑**：
1. 计算每张卡的平均交易金额和标准差
2. 统计一周内的消费城市和国家数量
3. 定义异常类型判断规则
4. 筛选满足任一异常条件的交易
5. 关联客户信息输出结果

```sql
WITH transaction_stats AS (
    SELECT 
        card_number,
        AVG(amount) AS avg_amount,
        STDDEV(amount) AS std_amount,
        COUNT(*) AS trans_count
    FROM credit_card_transactions
    WHERE transaction_date >= DATE_SUB(CURRENT_DATE, INTERVAL 3 MONTH)
    GROUP BY card_number
),
location_stats AS (
    SELECT 
        card_number,
        COUNT(DISTINCT city) AS city_count,
        COUNT(DISTINCT country) AS country_count
    FROM credit_card_transactions
    WHERE transaction_date >= DATE_SUB(CURRENT_DATE, INTERVAL 1 WEEK)
    GROUP BY card_number
)
SELECT 
    t.card_number,
    c.customer_name,
    t.avg_amount,
    t.std_amount,
    l.city_count,
    l.country_count,
    CASE 
        WHEN t.std_amount > t.avg_amount * 3 THEN '金额波动异常'
        WHEN l.city_count > 3 THEN '多地消费异常'
        WHEN l.country_count > 1 THEN '跨国消费异常'
        ELSE '其他异常'
    END AS risk_type
FROM transaction_stats t
JOIN location_stats l ON t.card_number = l.card_number
JOIN customers c ON t.card_number = c.card_number
WHERE t.trans_count >= 5
AND (t.std_amount > t.avg_amount * 3 OR l.city_count > 3 OR l.country_count > 1);
```
